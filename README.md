# astrbot_plugin_context_plus

超长记忆上下文增强插件 - 实现低成本、高缓存命中率的群聊长记忆系统

## 核心特性

### 🧠 三层记忆架构

传统上下文插件只保留最近几条消息，AI 无法记住几天前的对话内容。本插件通过**三层记忆架构**实现真正的长记忆：

| 层级 | 文件 | 内容 | 更新频率 |
|------|------|------|----------|
| **第一层** | `_summary.log` | 历史摘要（智能过滤日常吹水，保留重要信息） | 每天凌晨更新 |
| **第二层** | `_profile.md` | 群成员画像（好感度、称呼、性格等长期信息） | 每天凌晨更新 |
| **第三层** | `YYYY-MM-DD.log` | 详细聊天日志（按日期拆分，按预算加载） | 每天凌晨更新 |

**摘要过滤规则**：
- ❌ 过滤日常打招呼（早安/晚安/吃了没）
- ❌ 过滤表情包、纯水聊
- ✅ 保留有实质内容的讨论
- ✅ 保留决策、问题解答、重要事件

### 🚀 DeepSeek 缓存优化

DeepSeek 提供 **Prefix Caching** 功能，可以缓存 prompt 的前缀部分，大幅降低 token 成本。本插件专门针对此功能进行了架构优化。

#### 分层注入策略

```
system_prompt 结构（从上到下）：
┌────────────────────────────────────────────────────┐
│ 人设系统（由 AstrBot 管理）                          │ ← 不受插件影响
├────────────────────────────────────────────────────┤
│ <historical_summary>                               │ ← 历史摘要（天级变化）
│ </historical_summary>                              │
├────────────────────────────────────────────────────┤
│ <group_profile>                                    │ ← 群成员画像（天级变化）
│ </group_profile>                                   │
├────────────────────────────────────────────────────┤
│ <chat_logs_2024-01-15>                             │ ← 历史日志（天级变化）
│ </chat_logs_2024-01-15>                            │
├────────────────────────────────────────────────────┤
│ <chat_logs_2024-01-17>                             │ ← 当天日志（追加模式）
│   最新聊天记录...                                   │   前面内容不变，只有末尾增长
│ </chat_logs_2024-01-17>                            │
└────────────────────────────────────────────────────┘
```

#### 缓存命中率分析

由于当天日志采用**追加模式**，第 N 次请求的前缀与第 N-1 次请求高度匹配：

| 场景 | 缓存命中率 | 说明 |
|------|------------|------|
| 当天首次请求 | 0% | 无缓存，需要完整计算 |
| 当天第二次请求及之后 | **95-97%** | 前缀与上一次请求高度匹配，仅末尾少量新增内容需要重新计算 |
| 跨天后首次请求 | 0% | 前缀变化（摘要/画像更新） |
| 跨天后第二次请求 | 95-97% | 立即恢复高命中率 |

**关键发现**：由于当天日志采用追加模式，第 N 次请求的前缀与第 N-1 次请求高度匹配，因此**除首次请求外，后续请求的缓存命中率可达 95-97%**。

**影响因素**：
- 短期上下文变化：每次请求的短期上下文可能略有不同（-1~3%）
- 当天日志增长：活跃群聊日志增长更快（-0.5~2%）
- 其他插件影响：其他插件可能修改system_prompt（-0.1~0.5%）
- DeepSeek缓存策略：缓存限制和匹配策略（-1~2%）

**实际测试建议**：建议在实际部署后通过Token统计功能验证真实命中率。

#### 缓存优化策略总结

**核心思想**：将 prompt 内容按稳定性分层排列，最稳定的内容放在最前面（作为前缀），变化的内容放在末尾，最大化 DeepSeek Prefix Caching 的命中率。

本插件通过三层记忆架构 + 分层注入策略 + 当天日志追加模式，使第 N 次请求的前缀与第 N-1 次请求完全相同，实现 **95-97%** 的缓存命中率（除首次请求外）。

## 配置项

### 长记忆相关配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `chat_log_enabled` | bool | true | 是否启用聊天日志文件 |
| `chat_log_max_chars` | int | 16000 | 历史日志最大字符数（超出后只加载摘要） |
| `chat_log_days` | int | 5 | 保留完整日志的天数 |
| `auto_cleanup_old_logs` | bool | true | 是否自动删除超过 `chat_log_days` 的旧日志文件 |
| `summary_enabled` | bool | true | 是否启用每日摘要 |
| `summary_provider_id` | str | "" | 摘要专用 LLM 提供商（推荐使用主模型以共享缓存） |
| `summary_max_chars` | int | 10000 | 摘要文件最大字符数（超出后自动压缩） |
| `profile_enabled` | bool | true | 是否启用群成员画像 |

### 基础配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled_groups` | list | [] | 启用插件的群组ID（空=全部） |
| `recent_chats_count` | int | 15 | 短期上下文消息数量 |
| `max_images_in_context` | int | 4 | 上下文中最大图片数量 |
| `token_stats_enabled` | bool | true | 是否启用 Token 统计日志 |

## 命令

| 命令 | 说明 |
|------|------|
| `/reset` 或 `/new` | 清空当前群聊的上下文缓存 |
| `/profile` | 查看当前群聊的群成员画像 |
| `/profile_update` | 让 AI 分析聊天记录，更新群成员画像 |

## 数据存储结构

```
data/plugin_data/astrbot_plugin_context_plus/
├── context_cache.json          # 短期上下文缓存
└── chat_logs/
    └── {group_id}/
        ├── _summary.log        # 历史摘要（长期记忆）
        ├── _profile.md         # 群成员画像
        ├── 2024-01-15.log      # 按日期存储的聊天日志
        ├── 2024-01-16.log
        └── 2024-01-17.log
```

**自动清理**：超过 `chat_log_days` 的日志文件会在摘要生成后自动删除，只保留 `_summary.log` 和 `_profile.md`。
